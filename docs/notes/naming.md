# Naming: aide-de-camp / adc

## The Name

**aide-de-camp** (French: *aide de camp*, "camp assistant")

A military aide-de-camp has full operational context across all theaters, routes communications, handles any category of task — research, logistics, coordination, personal matters — and is specifically attached to *one* commanding officer, not a generic service.

This captures what the system is: not a coding assistant, not a voice interface, not a routing layer — a universal personal interface that handles anything the user asks, knows their full operational context, and improves through use.

## CLI Invocation

The CLI uses `adc` — the standard abbreviation for aide-de-camp.

```bash
adc dispatch "has the options pipeline caught up? also check the ibkr mcp status"
adc ask "what were the name candidates for pdftract?"
adc status
```

`adc` is short enough to type without friction, unambiguous in context, and carries the name forward without spelling it out on every invocation.

## Naming Process

Rejected directions:
- Acronyms (PRISM, LOOM, etc.) — don't impress or interest others
- Generic AI terms (assistant, agent, copilot) — indistinguishable in the current landscape
- Mythological figures — feel unoriginal, hard to own in search
- Taken names: Loom (Atlassian video product), Relay (Meta GraphQL client + relay.app), Helm (Kubernetes package manager), Shuttle (Shuttle.rs Rust hosting)

Considered:
- **Factotum** — Latin "do everything"; most semantically accurate; slightly hard to spell
- **Familiar** — the serving spirit with intimate knowledge of its master; evocative but niche
- **Majordomo** — chief household manager; good, but longer and the email list software lingers

**aide-de-camp** won because it combines operational precision with personal attachment — not a tool, not a service, but a specific individual assignment to one person.
