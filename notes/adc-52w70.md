## Implementation Details

### Result Type Format
```
{intent_type}:{project_slug}          # Intent-derived
lookup:{lookup_kind}:{project_slug}   # Lookup threads
monitoring:{project_slug}              # Monitoring results
{intent_type}:general                 # Fallback when no project
```

### Example Patterns
```python
# Status - High specificity
("status:k8s", "comp-k8s-pod-status", "normal", 0.94, 30)
("status:ci", "comp-ci-pipeline-status", "normal", 0.96, 28)

# Lookup - Logs need expanded view
("lookup:logs:k8s", "comp-k8s-logs", "expanded", 0.97, 50)

# Action - Deployment results
("action:deploy", "comp-deployment-result", "normal", 0.92, 25)

# Monitoring - K8s needs detailed view
("monitoring:k8s", "comp-k8s-monitoring", "expanded", 0.96, 25)

# Fallback - Generic components
("status:general", "comp-generic-status", "normal", 0.70, 30)
```

### Match Score Rationale
- **0.95-1.00**: Perfect - exact component designed for result type
- **0.85-0.94**: Excellent - very strong semantic match
- **0.70-0.84**: Good - strong match, general-purpose component fits
- **Below 0.70**: Falls back to generic card (below threshold)

### Sample Count Guidance
- Production usage: 20-50 (well-established patterns)
- Common patterns: 10-25 (frequently used)
- New patterns: 5-15 (recently established)
- Generic fallbacks: 15-30 (high usage, low specificity)
