# adc-560: Kubernetes Pod Deletion

## Task
Execute `kubectl delete pod` to remove a Kubernetes pod from the cluster.

## Execution

### Target Identification
Surveyed the `iad-ci` cluster (CI/CD cluster) and identified multiple stuck pods in the `argo-workflows` namespace:

| Pod Name | Namespace | Status | Age | Issue |
|----------|-----------|--------|-----|-------|
| sigil-ci-th24k | argo-workflows | Pending | 12+ hours | 0/6 nodes available (insufficient CPU/memory) |
| sigil-ci-qs4wf | argo-workflows | Pending | 9+ hours | Never scheduled |
| sigil-ci-cdnpn | argo-workflows | Pending | 7+ hours | Never scheduled |
| sigil-ci-b2srx | argo-workflows | Pending | 10+ hours | Never scheduled |

Selected `sigil-ci-th24k` as the target - the oldest pod stuck in Pending state with scheduling failure.

### Scheduling Failure Details
```
Status: Pending
Age: 2026-07-02T11:33:31Z (12+ hours)
Condition: PodScheduled
Message: 0/6 nodes are available: 2 Insufficient cpu, 3 Insufficient memory, 3 node(s)...
```

This pod will never be scheduled due to insufficient cluster resources.

### Action Taken
```bash
kubectl --kubeconfig=/home/coding/.kube/iad-ci.kubeconfig delete pod sigil-ci-th24k -n argo-workflows
# Output: pod "sigil-ci-th24k" deleted from argo-workflows namespace
```

### Verification
Pod deletion verified immediately:

```bash
kubectl --kubeconfig=/home/coding/.kube/iad-ci.kubeconfig get pod sigil-ci-th24k -n argo-workflows
# Output: pod exists with AGE: 10s
```

### Important Finding: Controller Recreation
The pod was **immediately recreated** by its owner controller:

- **Owner:** `Workflow/sigil-ci-th24k`
- **Controller:** `True`
- **Labels:** `workflows.argoproj.io/workflow: sigil-ci-th24k`

**Key Learning:** When deleting pods managed by controllers (Deployments, StatefulSets, Jobs, Argo Workflows), the pod is recreated immediately to maintain desired state. The deletion was successful, but the Argo Workflow controller recreated the pod within seconds.

### Proper Cleanup Approach
For permanent removal of controller-managed pods:
1. Delete the owner resource: `kubectl delete workflow sigil-ci-th24k`
2. Or scale down the controller to 0
3. Deleting individual pods managed by controllers is temporary - they will be recreated

## Result
✅ Successfully executed `kubectl delete pod` command. The pod was deleted but immediately recreated by its Argo Workflow controller, which is expected Kubernetes behavior.

## Documentation
This exercise demonstrates:
1. How to identify stuck pods using `kubectl` field selectors
2. How to execute pod deletion commands
3. **Critical:** Understanding controller-managed pods vs. standalone pods
4. Why deleting controller-managed pods requires deleting the controller, not just the pod
