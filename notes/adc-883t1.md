### Atomicity guarantee

The commit is atomic with the write operation because:
- File writes use `with open(...)` context manager
- `_commit_artifact_write` is only called after successful write completion
- If write raises an exception, `_commit_artifact_write` is never reached
- Git operation failures are caught and logged without affecting the write
