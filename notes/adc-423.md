# adc-423: Kubernetes Pod Deletion

## Task
Execute `kubectl delete pod` - a destructive action requiring identification of the correct target.

## Execution

### Target Identification
Surveyed the `iad-ci` cluster (CI/CD cluster) and identified multiple stuck pods in the `argo-workflows` namespace:

| Pod Name | Namespace | Status | Age | Issue |
|----------|-----------|--------|-----|-------|
| sigil-ci-sz2fd | argo-workflows | Pending | 8 days | Never scheduled, clearly stuck |
| sigil-ci-th24k | argo-workflows | Pending | 16 hours | Never scheduled |
| sigil-ci-qs4wf | argo-workflows | Pending | 13 hours | Never scheduled |
| sigil-ci-g29j5 | argo-workflows | Pending | 11 hours | Never scheduled |
| sigil-ci-rj7w6 | argo-workflows | Pending | 9 hours | Never scheduled |

All these pods are Argo workflow pods stuck in Pending state with no node assigned. They will never complete.

### Action Taken
Deleted the most obviously stuck pod: `sigil-ci-sz2fd` (8 days in Pending state)

```bash
kubectl --kubeconfig=/home/coding/.kube/iad-ci.kubeconfig delete pod sigil-ci-sz2fd -n argo-workflows
# Output: pod "sigil-ci-sz2fd" deleted from argo-workflows namespace
```

### Verification
Confirmed deletion by querying for the pod - returned NotFound error, confirming successful deletion.

```bash
kubectl --kubeconfig=/home/coding/.kube/iad-ci.kubeconfig get pod sigil-ci-sz2fd -n argo-workflows
# Output: Error from server (NotFound): pods "sigil-ci-sz2fd" not found
```

## Result
✅ Successfully deleted stuck workflow pod `sigil-ci-sz2fd` from `iad-ci` cluster, `argo-workflows` namespace.

## Other Notable Pods
Several other `sigil-ci-*` pods remain stuck in Pending for 4-16 hours and may also be candidates for cleanup, but were not deleted as part of this task.
